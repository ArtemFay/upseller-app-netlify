# Бэкап GAS-проекта CalendOTG

Эта папка — полная копия исходного Google Apps Script проекта "Календарь отгрузок" на момент начала миграции на веб-приложение (17 апреля 2026).

## Зачем

Текущий CalendOTG переводится на стек Next.js + Supabase (см. [../MIGRATION_PLAN.md](../MIGRATION_PLAN.md)). Файлы из этой папки сохранены, чтобы:

- в любой момент можно было вернуться к разработке на GAS;
- при переносе логики в новый стек сверяться с оригиналом как с эталоном;
- сохранить историю — что именно работало до миграции.

## Содержимое

| Файл | Назначение |
|---|---|
| `Main.gs` | Бэкенд — три endpoints (`getAppBootstrap`, `getCalendarData`, `updateShipmentRow`) + утилиты |
| `Index.html` | Фронтенд — SPA на Tabulator 6.3.0 с toolbar, вкладками дат, редактируемыми ячейками |
| `appsscript.json` | Манифест GAS (OAuth scopes, версия runtime) |
| `.clasp.json` | Конфигурация clasp: привязка к deployment ID `11xlzFVg1ZIh8rfdSAJ9p9ef0yIdQw7KHpqiEwzkek1lk-K7uGqq9D3ED` |
| `logotip  — копия.svg` | Логотип приложения |
| `Пример листа 🗓️ КЛД.csv` | Образец витрины-календаря (FILTER-формулы) |
| `Соответствие столбцов Отгрузки - Календарь.csv` | Маппинг столбцов между листами `ОТГ_FILT` и `🚚 ОТГ` |

## Как вернуться к работе на GAS

Если понадобится продолжить разработку GAS-версии:

1. Из корня проекта те же файлы доступны как рабочие (эта папка — копия).
2. Или перейти в `GAS/` и запускать `clasp push` отсюда — deployment ID тот же.
