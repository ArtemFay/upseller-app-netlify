# Приемка — web-модуль приёмки товара

Web-модуль приёмки товара для сайта Upseller. Модуль переносит рабочую логику из Google Apps Script «Приемка 2.0» в обычное web-приложение, которое развивается и деплоится вместе с общим сайтом.

## Текущий статус

- Стадия: интерактивный прототип, web-копия активного GAS-проекта.
- Цель ближайшего этапа: повторить полезную бизнес-логику актуального Google Apps Script, затем перестроить UX под best practice fulfillment-приёмки.
- Рабочие устройства: desktop 4:3, планшет 11", телефон, ТСД Urovo T40.
- Основной сценарий: выбрать поставку → загрузить состав → автоматически подготовить 9 закрытых коробов → открыть один активный короб сканом/тапом → наполнять сканом или ручным количеством → закрыть короб → перейти к следующему.
- При первом открытии страницы поставка не выбирается автоматически, короба и состав не создаются до выбора поставки.

## Локальный запуск

Через единый сервер сайта (рекомендуется — даёт ту же среду, что и прод):

```bash
cd ../
npm install            # один раз
npm run dev            # порт 3010, AUTH_DISABLED=true
```

Открыть `http://localhost:3010/PRIEMKA/`. Для планшета в одной Wi-Fi-сети — IP компьютера на месте `localhost`.

Важно: после переноса выбора поставки и bootstrap-данных приемка требует Express API:

- `GET /api/receiving/supplies`;
- `GET /api/receiving/bootstrap`.

Отдельный статический сервер (`python -m http.server`) больше подходит только для просмотра верстки без данных: рабочая приемка на планшете должна открываться через общий сервер `:3010`.

## Источники логики

- Актуальный GAS-черновик: `wms-receiving-gas-latest` в исторической рабочей папке.
- Константы и бизнес-процессы: [`1_CONST/`](../../../1_CONST/).
- Анализ переноса: [`docs/gas-v06-analysis-report.md`](docs/gas-v06-analysis-report.md).

Исторические GAS-файлы не переносятся сюда как рабочий код — они остаются источником анализа.
